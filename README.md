# gitlab-create-merge-request

## Overview

Create a GitLab Merge Request using the `python-gitlab` CLI (`gitlab` command).

If the user does not provide a title, the title is automatically generated by analyzing the full diff between the current branch and the target branch.

## When to Use

Use this skill when the user asks to:

- create a Merge Request
- open an MR
- submit a branch for review
- create a GitLab MR

Examples:

- "Create a merge request"
- "Open an MR for this branch"

## Requirements

The following tools must be installed:

- git
- python-gitlab CLI (`gitlab`)
- GitLab authentication configured

Example config location (Windows):

C:\Users\<username>\.python-gitlab.cfg

The following command must work:

gitlab project-merge-request create --help

The current directory must be a git repository.

## Inputs

Optional:

- title
- description
- target_branch

If `title` is not provided, generate it automatically from the diff.

If `target_branch` is not provided, detect it automatically.

## Workflow

### 1 Detect current branch

git rev-parse --abbrev-ref HEAD

Store as:

source_branch

### 2 Ensure branch is pushed

git push -u origin <source_branch>

### 3 Determine GitLab project path

git remote get-url origin

Example:

git@gitlab.com:group/project.git

Extract:

group/project

Use as:

project_id

### 4 Detect target branch

Try to detect the default branch:

git symbolic-ref refs/remotes/origin/HEAD

Example:

refs/remotes/origin/main

Extract:

main

If detection fails:

- use `main` if it exists
- otherwise use `master`

### 5 Generate MR title from diff

Get the full diff:

git diff <target_branch>...HEAD

Analyze the changes and generate a concise title describing the overall purpose of the changes.

Guidelines:

- Summarize the main purpose of the changes
- Prefer action-oriented titles
- Keep the title short (max ~72 characters)
- Use conventional commit style if possible

Examples:

feat: add OAuth login support  
fix: resolve login validation bug  
refactor: simplify authentication middleware

### 6 Create the Merge Request

gitlab project-merge-request create ^
  --project-id "<project_path>" ^
  --source-branch "<source_branch>" ^
  --target-branch "<target_branch>" ^
  --title "<generated_title>" ^
  --description "<description>"

## Output

Return:

- Merge Request URL
- Merge Request ID
- generated title

## Safety Rules

- Never expose authentication tokens
- Never force push unless explicitly requested
- Stop if the repository is not hosted on GitLab

## Example

Detected branch:

feature/oauth-login

Detected target branch:

main

Diff analyzed:

git diff main...HEAD

Generated title:

feat: add OAuth login support

Command executed:

gitlab project-merge-request create ^
  --project-id "group/project" ^
  --source-branch "feature/oauth-login" ^
  --target-branch "main" ^
  --title "feat: add OAuth login support"
